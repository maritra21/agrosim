## Step-by-Step Process

### Phase 1: Setup & Approval

**Week 1 - Farmer Registration**
1. Abdul Karim visits AgroConnect website
2. Clicks "Register" and selects "Farmer"
3. Fills in details:
   - Name: Abdul Karim
   - Village: Kumira
   - Phone: 01712345678
   - Address: Details of farm location
4. Submits registration
5. Gets message: "Pending approval"

**Week 1 - Admin Approval**
6. Admin logs in to control tower
7. Reviews Abdul Karim's registration
8. Verifies details look legitimate
9. Clicks "Approve"
10. Abdul Karim receives notification

**Week 1 - Buyer Registration**
11. Shahed Alam registers as "Buyer"
12. Provides cafe details in GEC Circle
13. Admin reviews and approves
14. Shahed receives confirmation

### Phase 2: Product Listing

**Day 1 Morning**
1. Abdul Karim logs into farmer dashboard
2. Clicks "Add Product"
3. Enters product details:
   - Name: Fresh Tomatoes
   - Category: Vegetables
   - Description: Freshly harvested this morning
   - Price: 50 BDT per kg
   - Available: 100 kg
   - Harvest Date: Today
4. Submits the product
5. Product now visible to all buyers

**Day 1 - Multiple Products**
6. Karim adds more products:
   - Cucumbers: 30 BDT/kg, 80 kg
   - Leafy Greens: 25 BDT/kg, 50 kg
7. His digital stall is now fully stocked

### Phase 3: Browsing & Ordering

**Day 1 Afternoon**
1. Shahed Alam logs into buyer dashboard
2. Sees all available products
3. Uses search to find "tomatoes"
4. Sees Abdul Karim's listing:
   - Fresh Tomatoes
   - 50 BDT/kg
   - Farmer: Abdul Karim
   - Village: Kumira
   - Available: 100 kg
5. Decides to order 20 kg
6. Enters quantity: 20
7. Clicks "Add to Cart"

**Day 1 - Shopping Cart**
8. Also adds:
   - Cucumbers: 10 kg
   - Leafy Greens: 5 kg
9. Reviews cart:
   - Tomatoes: 20 kg × 50 = 1000 BDT
   - Cucumbers: 10 kg × 30 = 300 BDT
   - Leafy Greens: 5 kg × 25 = 125 BDT
   - **Total: 1425 BDT**
10. Enters delivery address:
    - Shahed's Cafe, GEC Circle
11. Preferred delivery: Tomorrow morning
12. Clicks "Place Order"

### Phase 4: Order Fulfillment

**Immediate - Notifications**
1. Abdul Karim's phone buzzes
2. Notification: "New Order Received!"
3. Order details:
   - Buyer: Shahed Alam
   - Amount: 1425 BDT
   - Items: Tomatoes (20kg), Cucumbers (10kg), Leafy Greens (5kg)
   - Delivery: GEC Circle
4. Karim logs in to see full details

**Day 1 Evening - Confirmation**
5. Karim reviews the order
6. Checks he has the products ready
7. Clicks "Confirm Order"
8. Shahed receives notification:
   - "Your order has been confirmed!"
   - Expected delivery: Tomorrow morning

**Day 2 Morning - Delivery**
9. Karim harvests fresh products
10. Packs them carefully
11. Delivers to Shahed's Cafe in GEC Circle
12. Shahed inspects and accepts delivery
13. Karim logs in and marks "Delivered"
14. Shahed receives notification: "Order Delivered"

### Phase 5: Tracking & Analytics

**Farmer Dashboard (Karim)**
- Total Products: 3

**Buyer Dashboard (Shahed)**

**Admin Dashboard**

**Admin Broadcast Example**

## Technical Flow

```
User Action → Frontend (HTML/JS) → API Request → Backend (Node.js)
                                                      ↓
                                                 Authenticate (JWT)
                                                      ↓
                                                 Authorize (Role)
                                                      ↓
                                                 Process (Business Logic)
                                                      ↓
                                                 Database (MySQL)
                                                      ↓
                                                 Return Response
                                                      ↓
Frontend (Display) ← JSON Response ←─────────────────┘
```

## Data Flow Example

**Placing an Order**:
1. Buyer selects products → Cart in browser memory
2. Click "Place Order" → POST /api/orders
3. Backend receives → Validates JWT token
4. Checks role → Must be "buyer"
5. Verifies products → Stock available?
6. Creates order record → MySQL INSERT
7. Updates product quantities → MySQL UPDATE
8. Creates notification → For farmer
9. Returns success → Order ID
10. Frontend shows → "Order placed successfully!"
